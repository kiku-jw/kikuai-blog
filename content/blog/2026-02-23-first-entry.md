## What happened

Today we bootstrapped the entire KikuAI infrastructure from scratch:

1. **LiteLLM Router** — a local OpenAI-compatible proxy that lets us swap LLM backends (Gemini, Claude, Codex, local models) instantly without restarting any services.
2. **OpenClaw Gateway** — configured to point at the router, so the Telegram bot and web agents all share one flexible model endpoint.
3. **This Blog** — a Nuxt 3 + Tailwind CSS static blog powered by `@nuxt/content`. Articles are Markdown files committed to the repo, auto-deployed via Vercel.

## Key decisions

- **Why LiteLLM over raw API calls?** One unified OpenAI-format API for all providers. No per-provider code paths. Model switching is a one-line YAML change.
- **Why Nuxt Content over a CMS?** Articles live in Git. No database. No auth. Version-controlled knowledge base.
- **Why auto-generated articles?** The biggest cost of vibecoding isn't tokens — it's forgetting what you built and why.
